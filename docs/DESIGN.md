# 墨笺全站设计规范

## 1. 设计定位

墨笺是以中文阅读为核心的个人博客。所有路由共享同一水墨视觉外壳；视觉语言来自宣纸、墨色、朱砂印章、山水长卷与书斋器物。页面应安静、克制、留白充足，避免现代卡片模板感。

设计原则：

- 内容优先：标题、摘要、日期与分类先于装饰。
- 水墨统一：颜色、字体、纹理、线条和插画保持同一东方书写语境。
- 布局借鉴，样式独立：可学习其他博客的信息架构，不复制其卡片、配色、字体和素材。
- 动效轻量：以进入与定位动画为主；全局氛围动效（花瓣、墨点、墨晕）收敛在单个画布与单一 `requestAnimationFrame` 内，`prefers-reduced-motion` 时整体关闭。
- 响应式降级自然：宽屏使用侧栏，窄屏恢复线性阅读顺序。

## 2. 页面骨架

全站通过 `BaseLayout` 共用文档结构、固定导航、页脚、色彩与排版。首页在共享外壳内使用水墨首屏与内容网格；归档、分类、寻墨、留墨、关于、文章详情和 404 保持适合自身内容的阅读布局。

### 宽屏布局

适用条件：视口宽度不小于 `1440px`。

```text
┌──────────────────────────────────────────────┐
│                  水墨首屏                    │
└────────────── 内容间隔 24px ─────────────────┘
┌──────────┬──────────────────────┬──────────┐
│ 书斋目录 │      近来所记        │ 案头近况 │
│ 240–280  │      至少 620px      │ 260–300  │
├──────────┤                      │          │
│ 关于此间 │      卷轴文章流      │ 一页小记 │
└──────────┴──────────────────────┴──────────┘
```

布局参数：

| 项目 | 值 |
| --- | --- |
| 最大内容宽度 | `1480px` |
| 左侧栏 | `240–280px` |
| 中央内容 | `minmax(620px, 1fr)` |
| 右侧栏 | `260–300px` |
| 列间距 | `24px` |
| 首屏间距 | `24px` |

内容映射：

- 左侧上方：书斋目录与分类入口。
- 左侧下方：关于此间、长期记录主题和水墨竹影。
- 中央：近来所记、三篇卷轴文章和全部文章入口。
- 右侧：案头近况、当前状态和一页小记。

### 中屏布局

视口处于 `821–1439px` 时使用两列网格：书斋目录与关于此间位于左栏，近来所记位于右栏，案头近况横跨两栏置于下方。卷轴文章使用“元信息 + 正文”两列，题签落在正文下方。

### 小屏布局

视口不大于 `820px` 时保留单列顺序：

1. 水墨首屏
2. 书斋目录
3. 近来所记
4. 案头近况
5. 关于此间
6. 页脚

移动端不得产生横向滚动；卷轴文章、目录条目和案头信息在窄屏内改为单列。

## 3. 视觉系统

### 色彩

昼色（默认）：

| Token | 色值 | 用途 |
| --- | --- | --- |
| `--paper` | `#f2eee3` | 页面宣纸底色 |
| `--paper-light` | `#fbf9f3` | 浅色内容面 |
| `--paper-warm` | `#e8e0d1` | 暖纸层次 |
| `--paper-deep` | `#d8cdbc` | 深纸边缘 |
| `--ink` | `#292a27` | 主文字 |
| `--ink-soft` | `#66665f` | 正文与说明 |
| `--ink-faint` | `#929087` | 次要信息 |
| `--moss` | `#586258` | 山水与辅助强调 |
| `--moss-deep` | `#3f4841` | 深色交互状态 |
| `--cinnabar` | `#a44434` | 印章、序号与焦点 |
| `--cinnabar-deep` | `#7f3026` | 朱砂深色边缘 |
| `--petal` | `214 142 150` | 花瓣与墨点画布色（RGB 三元组） |

昼夜双主题：`html[data-theme="night"]` 整套切换为夜色——纸色转深（`#191a16` 系）、墨色转浅（`#e9e2d0` 系）、朱砂提亮（`--cinnabar: #d96b58`，保证暗底下小字对比度）、花瓣转暗（`176 108 116`）。完整夜值见 `global.css`。主题存于 `localStorage("mj-theme")`，由 `<head>` 内联脚本在首绘前应用到 `<html data-theme>`，不出现先昼后夜的闪烁。

避免引入高饱和渐变、纯黑背景或与现有暖灰体系冲突的冷色。

### 字体

- 文学性标题与正文：Noto Serif SC（400 / 500 / 700），宋体回退；`body` 使用 `data-font-role="serif"` 并默认继承，`--sans`（MiSans 系）仅作备用 token，不默认引用。
- 展示级书法标题（页面大标题、区块标题、手卷标题、印章文字、归档干支章）：Ma Shan Zheng（`@fontsource/ma-shan-zheng`），字重保持 400。
- 手写小字（kicker、眉批）：Long Cang（`@fontsource/long-cang`）。
- 字体角色协议为 `data-font-role="serif|cal|hand"`；字体分配唯一出处为 `src/styles/typography.css`，组件不得声明角色字体的 `font-family`。
- 大标题保持较松字距和紧凑行高；正文保持约 `1.8–2.15` 行高。
- `src/integrations/font-pipeline.mjs` 在 Astro 输出最终 HTML 后提取实际字符，按角色用 `subset-font` 裁切 Fontsource WOFF2 构建源，生成哈希文件、精确 `unicode-range`、清单与 OFL 许可证；HTML 仅注入本地字体 CSS 和当前页面必要的 preload，不依赖运行时字体 CDN。
- 正文字体子集覆盖全部页面文本，确保装饰字体缺字或加载失败时仍回退到 Noto；未知运行时文本继续回退到系统宋体栈。

### 形态与材质

- 主要边界使用细线、纸张色差和墨色浓淡，不使用通用圆角卡片。
- 首页主区块在中宽屏和宽屏使用 `16px` 轻圆角；内部内容保持开放，不叠加通用卡片容器。
- 文章使用横卷结构：卷轴、宣纸、朱砂卷标、日期栏和墨迹插画。
- 案头近况使用深墨绿色区域，形成全页唯一的明显明暗切换。
- 装饰应来自山水、竹影、印章、笔触和纸纹；Material Symbols 仅服务于共享导航的信息识别，不进入正文装饰。

## 4. 区块规范

### 顶部导航

- 固定在页面顶部。
- 左侧为“墨笺”字标与印章，右侧统一为墨庐、笺录、卷藏、此间、留墨，末端为昼夜切换钮（昼示「夜」、夜示「昼」，纯 CSS 切换显示）。
- 顶部导航是唯一玻璃控制层：使用半透明纸色、轻量背景模糊、同心胶囊圆角与内缘高光；滚动进入正文后提高纸色不透明度以保持可读性。
- 当前阅读区通过 `aria-current="page"` 标记。

### 水墨首屏

- 维持单一视觉焦点：“墨笺有余闲。”
- 桌面端接近完整视口高度，移动端根据内容自然增高。
- 左侧为标题、简介和入口；右侧为案上近况；底部为山景。
- 不增加轮播或视频背景。
- 首屏入场动画由 `body.loaded` 门控：loader 结束后播放；Swup 回访与无 JS 时内容直接显示。

### 书斋目录

- 四项分类：日常、行旅、读书、摄影。
- 使用序号、名称、描述和文章数量四层信息；数量由 `getPublishedPosts()` 按分类实时统计，不写死。
- 分类条目链接到对应 `category/<slug>/` 静态分类页。
- 宽屏侧栏内改为紧凑纵向排列；中小屏恢复完整横向信息结构。

### 近来所记

- 全页主内容，视觉权重最高。
- 每篇文章必须保留标题、摘要、日期、分类、阅读时长和标签。
- 整张手卷为单一链接进入文章详情，不嵌套第二个链接。
- 桌面端文章手卷统一高 `180px`，木轴宽 `12px`；移动端恢复自动高度并保持完整展开。
- 手卷默认轻微收拢，悬停或键盘聚焦时以 `520ms cubic-bezier(0.22, 1, 0.36, 1)` 向两侧展开；文字位置和卡片占位不得变化。

### 案头近况

- 展示正在做、正在读、正在学三种状态。
- 右侧栏内使用纵向信息行；单列布局下恢复更宽松的排列。
- 深色背景只用于此区块，避免在其他区块重复使用。

### 关于此间

- 说明博客记录范围和长期主题。
- 竹影、墨环和印章为辅助图形，不应压过文字。

### 归档年谱

- 归档页为年谱时间线：中央虚线贯穿，年份以干支圆章标记（Ma Shan Zheng、竖排），条目按年份分组并左右交替落在线两侧。
- 条目含日期、标题、分类；分类链接进入对应分类页。
- 视口不大于 `680px` 时时间线靠左，条目全部落在线右。

### 留墨

- 留墨页为访客书信入口：表单卡（称呼、留言）提交后经 `mailto:` 唤起本地信使，收件人为 `siteConfig.contactEmail`；提交后盖「已收到」印章并显示 toast。
- 页面下方预留 giscus 评论墙：`siteConfig.giscus` 的 `repoId` 与 `categoryId` 配好后才渲染，未配置不加载任何第三方脚本。

## 5. 响应式规则

| 断点 | 行为 |
| --- | --- |
| `≥1440px` | 三列首页网格，最大宽度 `1480px`，内容与首屏间隔 `24px` |
| `821–1439px` | 两列首页网格，案头近况下沉并横跨两栏 |
| `≤1080px` | 原有宽屏组件开始收缩，案头内容改单列 |
| `≤820px` | 首屏、目录、关于改为单列 |
| `≤680px` | 归档时间线靠左 |
| `≤620px` | 缩小页面边距，卷轴文章完全单列化 |

布局调整优先使用 CSS Grid；不得通过 JavaScript 判断视口并重排首页内容。

## 6. 动效与性能

- 首屏和滚动 reveal 动画约 `300ms`。
- 首载 loader：整页宣纸遮罩，「墨」「笺」逐字显现加墨晕，同时等待页面与 Noto Serif SC 400、Ma Shan Zheng 400 就绪，最短停留约 900ms；失败或慢网由 `2.6s` 硬超时释放。仅在 `.js` 环境渲染，不参与 Swup 交换。
- 全局氛围动效由 `src/components/layout/InkEffects.astro` 集中管理，仍只使用一个 `<canvas id="fx">` 与一个 `requestAnimationFrame`：花瓣保持轻缓；精细指针轨迹由外洇、墨身、积墨核与飞白组成，快速移动才带短暂轻烟；点击墨渍依次经历压墨、毛细扩散与积边干燥，并附少量定向墨滴。Canvas backing store 的 DPR 上限为 2，stamp / 烟丝 / 墨晕 / 墨滴分别硬封顶为 120 / 12 / 4 / 40；粗指针不显示移动轨迹，`prefers-reduced-motion` 时整体停止并清空。当前原生 Canvas 2D 足以完成合成、渐变、路径与离屏预渲染，不引入渲染依赖；只有需要持续流体模拟、自定义 GPU shader 或数百粒子时，才评估 GPU/流体库。
- 昼夜切换经 View Transitions 从新主题按钮中心圆形扩散：`600ms`、`ease-in-out`，圆心为按钮中心的文档坐标百分比、半径为覆盖可见视口四角的参考盒对角线分数（根捕获盒是整个文档而非视口，且随浏览器有 CSS/设备像素缩放，百分比与两者无关）；扩散期间经 `html.theme-vt` 关闭 body 颜色交叉淡化（防快照整页重栅格卡顿、保证边界锐利）；`prefers-reduced-motion` 或浏览器不支持时回退瞬时切换。
- 滚动进度条与回顶按钮仅使用 `opacity`/`transform`，不参与布局。
- 动画仅使用 `opacity` 与 `transform`；唯一例外是昼夜扩散在 `::view-transition-new(root)` 上动画化 `clip-path: circle()`。
- reveal 隐藏态由 `.js` 根类门控：禁用 JavaScript 时内容默认可见，题图默认可靠显示。
- 禁止无限山景漂移和固定全屏 SVG 噪点。导航 `backdrop-filter` 仅用于浮动控制层，禁止扩散到正文或动画化模糊值。
- 尊重 `prefers-reduced-motion: reduce`，关闭动画时内容必须直接可见。
- 不为全站动效新增依赖。
- 生产 WOFF2 总量不得超过 750 KiB，单文件不得超过 220 KiB；首页只预载 Noto Serif SC 400 的命中分片和 Ma Shan Zheng 命中分片，Long Cang 不预载。

## 7. 可访问性

- 使用 `header`、`nav`、`main`、`section`、`article`、`aside`、`footer` 等语义元素。
- 每个主要区块由 `aria-labelledby` 关联可见标题。
- 图片和纯装饰 SVG 明确区分；装饰内容使用 `aria-hidden="true"`。
- 键盘焦点使用朱砂色可见轮廓。
- 链接文案必须能独立说明目标，文章链接提供明确的 `aria-label`。
- 禁用 JavaScript 时核心内容完整可见：reveal 与题图默认显示，loader 与画布动效不渲染。
- 页面在 `390px` 宽视口下不得横向溢出。

## 8. 实现约束

- 所有页面必须使用 `src/layouts/BaseLayout.astro`；共享导航、页脚和视觉基础分别只在 `SiteHeader`、`SiteFooter` 与 `global.css` 维护。
- `src/pages/index.astro` 只维护首页专属首屏和内容区，不重复声明文档、导航、页脚或全局基础样式。
- 保持 GitHub Pages 的 `/mojian/` 基础路径兼容，站内链接继续使用现有 `BASE`。
- 不为纯布局或动画新增依赖；字体构建依赖只允许用于离线裁切，不得产生运行时服务或 CDN 请求。
- 修改布局时不得顺手更换配色、字体、纹理、文案和文章内容。
- 启动开发服务使用 `astro dev --background`，完成后使用 `astro dev stop`。

最低验证：

```bash
npm run check
npm run build
```

浏览器验证至少覆盖首页、归档、关于、文章详情和 404：

- 桌面 `1440 × 900`：三列、首屏重叠、文章跳转、无横向溢出。
- 中屏 `1280 × 900`：两列、案头近况横跨两栏、无断点拥挤。
- 手机 `390 × 844`：单列、首屏可读、文章卡片完整、无横向溢出。
- 控制台无相关 `error` 或 `warning`。

## 9. 参考与边界

首页宽屏信息架构参考 [CuteLeaf/Firefly](https://github.com/CuteLeaf/Firefly) 及其[线上示例](https://firefly.cuteleaf.cn/) 的“首屏横幅 + 左右侧栏 + 中央文章流 + 移动端侧栏下沉”思路。内页仅沿用墨笺的共享视觉外壳，不机械复制首页模块。

仅参考布局关系。墨笺不复制 Firefly 的圆角卡片、绿色主题、动漫素材、导航图标、组件代码和交互功能；最终视觉与内容体系保持独立。
