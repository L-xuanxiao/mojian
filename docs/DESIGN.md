# 墨笺全站设计规范

## 1. 设计定位

墨笺是以中文阅读为核心的个人博客。所有路由共享同一水墨视觉外壳；视觉语言来自宣纸、墨色、朱砂印章、山水长卷与书斋器物。页面应安静、克制、留白充足，避免现代卡片模板感。

设计原则：

- 内容优先：标题、摘要、日期与分类先于装饰。
- 水墨统一：颜色、字体、纹理、线条和插画保持同一东方书写语境。
- 布局借鉴，样式独立：可学习其他博客的信息架构，不复制其卡片、配色、字体和素材。
- 动效轻量：动画只辅助进入和定位，不持续占用绘制资源。
- 响应式降级自然：宽屏使用侧栏，窄屏恢复线性阅读顺序。

## 2. 页面骨架

全站通过 `BaseLayout` 共用文档结构、固定导航、页脚、色彩与排版。首页在共享外壳内使用水墨首屏与内容网格；归档、关于、文章详情和 404 保持适合自身内容的阅读布局。

### 宽屏布局

适用条件：视口宽度不小于 `1360px`。

```text
┌──────────────────────────────────────────────┐
│                  水墨首屏                    │
└────────────── 内容上移 72px ─────────────────┘
┌──────────┬──────────────────────┬──────────┐
│ 书斋目录 │      近来所记        │ 案头近况 │
│ 240–280  │      至少 620px      │ 260–300  │
├──────────┤                      │          │
│ 关于此间 │      卷轴文章流      │ 一页小记 │
└──────────┴──────────────────────┴──────────┘
```

布局参数：

| 项目 | 值 |
| --- | --- |
| 最大内容宽度 | `1480px` |
| 左侧栏 | `240–280px` |
| 中央内容 | `minmax(620px, 1fr)` |
| 右侧栏 | `260–300px` |
| 列间距 | `24px` |
| 首屏重叠 | `72px` |

内容映射：

- 左侧上方：书斋目录与分类入口。
- 左侧下方：关于此间、长期记录主题和水墨竹影。
- 中央：近来所记、三篇卷轴文章和全部文章入口。
- 右侧：案头近况、当前状态和一页小记。

### 中屏布局

视口处于 `821–1359px` 时使用两列网格：书斋目录与关于此间位于左栏，近来所记位于右栏，案头近况横跨两栏置于下方。卷轴文章使用“元信息 + 正文”两列，题签落在正文下方。

### 小屏布局

视口不大于 `820px` 时保留单列顺序：

1. 水墨首屏
2. 书斋目录
3. 近来所记
4. 案头近况
5. 关于此间
6. 页脚

移动端不得产生横向滚动；卷轴文章、目录条目和案头信息在窄屏内改为单列。

## 3. 视觉系统

### 色彩

| Token | 色值 | 用途 |
| --- | --- | --- |
| `--paper` | `#f2eee3` | 页面宣纸底色 |
| `--paper-light` | `#fbf9f3` | 浅色内容面 |
| `--paper-warm` | `#e8e0d1` | 暖纸层次 |
| `--paper-deep` | `#d8cdbc` | 深纸边缘 |
| `--ink` | `#292a27` | 主文字 |
| `--ink-soft` | `#66665f` | 正文与说明 |
| `--ink-faint` | `#929087` | 次要信息 |
| `--moss` | `#586258` | 山水与辅助强调 |
| `--moss-deep` | `#3f4841` | 深色交互状态 |
| `--cinnabar` | `#a44434` | 印章、序号与焦点 |
| `--cinnabar-deep` | `#7f3026` | 朱砂深色边缘 |

避免引入高饱和渐变、纯黑背景或与现有暖灰体系冲突的冷色。

### 字体

- 标题与文学性正文：`Source Han Serif SC`、`Noto Serif SC`、宋体回退。
- 导航与界面信息：`MiSans`、`Microsoft YaHei`、系统无衬线回退。
- 大标题保持较松字距和紧凑行高；正文保持约 `1.8–2.15` 行高。
- 不为布局参考更换字体或改变字体性格。

### 形态与材质

- 主要边界使用细线、纸张色差和墨色浓淡，不使用通用圆角卡片。
- 文章使用横卷结构：卷轴、宣纸、朱砂卷标、日期栏和墨迹插画。
- 案头近况使用深墨绿色区域，形成全页唯一的明显明暗切换。
- 装饰应来自山水、竹影、印章、笔触和纸纹；Material Symbols 仅服务于共享导航的信息识别，不进入正文装饰。

## 4. 区块规范

### 顶部导航

- 固定在页面顶部。
- 左侧为“墨笺”字标与印章，右侧统一为墨庐、笺录、卷藏、此间。
- 顶部导航是唯一玻璃控制层：使用半透明纸色、轻量背景模糊、同心胶囊圆角与内缘高光；滚动进入正文后提高纸色不透明度以保持可读性。
- 当前阅读区通过 `aria-current="page"` 标记。

### 水墨首屏

- 维持单一视觉焦点：“墨笺有余闲。”
- 桌面端接近完整视口高度，移动端根据内容自然增高。
- 左侧为标题、简介和入口；右侧为案上近况；底部为山景。
- 不增加轮播、视频背景或持续漂移动画。

### 书斋目录

- 四项分类：日常、行旅、读书、摄影。
- 使用序号、名称、描述和文章数量四层信息。
- 宽屏侧栏内改为紧凑纵向排列；中小屏恢复完整横向信息结构。

### 近来所记

- 全页主内容，视觉权重最高。
- 每篇文章必须保留标题、摘要、日期、分类、阅读时长和标签。
- 整个标题链接进入文章详情，不增加重复按钮。

### 案头近况

- 展示正在做、正在读、正在学三种状态。
- 右侧栏内使用纵向信息行；单列布局下恢复更宽松的排列。
- 深色背景只用于此区块，避免在其他区块重复使用。

### 关于此间

- 说明博客记录范围和长期主题。
- 竹影、墨环和印章为辅助图形，不应压过文字。

## 5. 响应式规则

| 断点 | 行为 |
| --- | --- |
| `≥1360px` | 三列首页网格，最大宽度 `1480px`，内容与首屏重叠 |
| `821–1359px` | 两列首页网格，案头近况下沉并横跨两栏 |
| `≤1080px` | 原有宽屏组件开始收缩，案头内容改单列 |
| `≤820px` | 首屏、目录、关于改为单列 |
| `≤620px` | 缩小页面边距，卷轴文章完全单列化 |

布局调整优先使用 CSS Grid；不得通过 JavaScript 判断视口并重排首页内容。

## 6. 动效与性能

- 首屏和滚动 reveal 动画约 `300ms`。
- 动画仅使用 `opacity` 与 `transform`。
- 禁止无限山景漂移和固定全屏 SVG 噪点。导航 `backdrop-filter` 仅用于浮动控制层，禁止扩散到正文或动画化模糊值。
- 尊重 `prefers-reduced-motion: reduce`，关闭动画时内容必须直接可见。
- 不为全站动效新增依赖。

## 7. 可访问性

- 使用 `header`、`nav`、`main`、`section`、`article`、`aside`、`footer` 等语义元素。
- 每个主要区块由 `aria-labelledby` 关联可见标题。
- 图片和纯装饰 SVG 明确区分；装饰内容使用 `aria-hidden="true"`。
- 键盘焦点使用朱砂色可见轮廓。
- 链接文案必须能独立说明目标，文章链接提供明确的 `aria-label`。
- 页面在 `390px` 宽视口下不得横向溢出。

## 8. 实现约束

- 所有页面必须使用 `src/layouts/BaseLayout.astro`；共享导航、页脚和视觉基础分别只在 `SiteHeader`、`SiteFooter` 与 `global.css` 维护。
- `src/pages/index.astro` 只维护首页专属首屏和内容区，不重复声明文档、导航、页脚或全局基础样式。
- 保持 GitHub Pages 的 `/mojian/` 基础路径兼容，站内链接继续使用现有 `BASE`。
- 不新增性能、布局或动画依赖。
- 修改布局时不得顺手更换配色、字体、纹理、文案和文章内容。
- 启动开发服务使用 `astro dev --background`，完成后使用 `astro dev stop`。

最低验证：

```bash
npm run check
npm run build
```

浏览器验证至少覆盖首页、归档、关于、文章详情和 404：

- 桌面 `1440 × 900`：三列、首屏重叠、文章跳转、无横向溢出。
- 中屏 `1280 × 900`：两列、案头近况横跨两栏、无断点拥挤。
- 手机 `390 × 844`：单列、首屏可读、文章卡片完整、无横向溢出。
- 控制台无相关 `error` 或 `warning`。

## 9. 参考与边界

首页宽屏信息架构参考 [CuteLeaf/Firefly](https://github.com/CuteLeaf/Firefly) 及其[线上示例](https://firefly.cuteleaf.cn/) 的“首屏横幅 + 左右侧栏 + 中央文章流 + 移动端侧栏下沉”思路。内页仅沿用墨笺的共享视觉外壳，不机械复制首页模块。

仅参考布局关系。墨笺不复制 Firefly 的圆角卡片、绿色主题、动漫素材、导航图标、组件代码和交互功能；最终视觉与内容体系保持独立。
